# 법령 문서 등록 가이드

## 지원 파일

| 형식 | 지원 여부 | 비고 |
|---|---|---|
| PDF (`.pdf`) | 지원 | `pdf-parse`로 텍스트 추출. 스캔 이미지 PDF는 텍스트가 없어 등록이 거부됨(아래 참고) |
| Word (`.docx`) | 지원 | `mammoth`로 텍스트 추출 |
| 텍스트 (`.txt`) | 지원 | 원문 그대로 사용 |
| 마크다운 (`.md`) | 지원 | 원문 그대로 사용 |
| 한글(`.hwp`, `.hwpx`) | **미지원** | 업로드 시 "PDF, DOCX 또는 TXT로 변환하여 등록해 주세요" 안내가 표시됨 |
| 실행 파일(`.exe`, `.sh` 등) | 차단 | 업로드 자체가 거부됨 |

파일 크기 상한은 `MAX_UPLOAD_SIZE_MB` 환경변수(기본 20MB)로 관리한다.

## 법령 문서 등록 방법

1. `/admin/documents/new`로 이동 (ADMIN 권한 필요)
2. 다음 정보를 입력한다.
   - **문서명**: 정식 명칭 (예: "도시 및 주거환경정비법")
   - **약칭**: 화면에 짧게 표시할 이름(선택)
   - **문서 종류**: 법률/시행령/시행규칙/조례/조례 시행규칙/고시·지침/법령해석/유권해석/
     판례/행정심판/내부 검토자료/기타 중 선택
   - **관할 구분/관할 지역**: 아래 "지역 설정" 참고
   - **사업 유형**: 재개발/재건축/가로주택정비/소규모재건축/소규모재개발/자율주택정비/
     모아타운/기타 중 해당하는 것(복수 선택 가능). 특정 사업 유형에 한정되지 않으면
     선택하지 않아도 된다(모든 사업 유형 질문에서 검색 대상이 됨).
   - **발령기관**, **원문 URL**, **설명**(모두 선택)
   - **버전 라벨**, **공포일**, **시행 시작일**, **시행 종료일**, **현행 버전으로 지정** 여부
   - **파일**
3. 등록을 누르면 즉시 텍스트 추출·조문 파싱·청크 생성이 진행되며, 완료되면 문서 상태가
   자동으로 `ACTIVE`가 된다(오류 시 `ERROR`, 처리 이력은 문서 상세 화면에서 확인).

## 시행일 입력 기준

- **공포일**: 법령·조례가 공포된 날짜(참고용, 검색 필터에는 사용하지 않음).
- **시행 시작일(effectiveFrom)** / **시행 종료일(effectiveTo)**: 이 버전이 실제로 효력을
  가지는 기간. 사용자가 챗봇에서 기준일을 선택하면, 그 날짜가 이 범위 안에 있는 버전만
  검색 대상이 된다.
- 시행 종료일을 비워두면 "현재까지 유효"로 해석된다(`effectiveTo = null`).
- 개정으로 새 버전을 등록할 때는 이전 버전의 시행 종료일을 새 버전의 시행 시작일
  전날로 맞추는 것을 권장한다(겹치거나 비는 기간이 없도록).
- **현행 여부(isCurrent)**: 기준일을 지정하지 않고 질문했을 때 기본으로 검색되는 버전.
  새 버전을 "현행"으로 등록하면 기존 현행 버전은 자동으로 현행 해제된다.

## 지역 설정

- **전국 공통 법령**(법률·시행령·시행규칙 등)은 관할 구분을 "전국 공통"으로, 관할 지역은
  "전국"으로 입력한다. 전국 공통 법령은 사용자가 어떤 지역을 선택하든 항상 검색된다.
- **광역자치단체 조례**(서울특별시, 부산광역시 등)는 관할 구분을 "광역자치단체"로,
  관할 지역에 정확한 광역자치단체명을 입력한다.
- **기초자치단체 조례**(예: 서울특별시 강남구)는 관할 구분을 "기초자치단체"로, 관할
  지역에 "서울특별시 강남구"처럼 상위 광역명을 포함해 입력하는 것을 권장한다(검색 시
  상위 광역 선택과 부분 일치로 매칭됨).
- **서울특별시 조례는 부산광역시를 선택한 질문에 자동으로 적용되지 않는다.** 사용자가
  지역을 선택하지 않거나 "전국"을 선택하면 전국 공통 법령만 검색되고, 특정 지역 조례는
  검색되지 않는다.

## 스캔 PDF 처리

텍스트 레이어가 없는 스캔 이미지 PDF를 업로드하면, 시스템은 추측으로 내용을 만들어내지
않고 다음과 같이 처리한다.

- `DocumentVersion.parsingStatus = NO_TEXT_EXTRACTED`
- `LegalDocument.status = ERROR`
- 화면 안내: "텍스트를 추출할 수 없는 스캔 문서입니다. OCR 처리 또는 텍스트 변환본 등록이
  필요합니다."

OCR은 이번 MVP 범위에 포함되지 않는다(`docs/NEXT_STEPS.md` 참고). 스캔 문서는 OCR 도구로
텍스트를 추출한 뒤 TXT/DOCX로 재등록해야 한다.

## 문서 버전 관리

- 법령이 개정되면 **기존 버전을 수정하지 않고** 문서 상세 화면의 "새 버전 등록"으로 새
  DocumentVersion을 추가한다.
- 동일한 파일(내용이 완전히 같은 파일)은 `contentHash` 중복 검사로 재등록이 차단된다.
- "재처리" 버튼은 같은 버전의 원본 파일을 다시 파싱해 LegalChunk를 재생성한다(파싱 로직
  개선 후 기존 문서에 반영할 때 사용). DocumentVersion 자체나 원본 파일은 바뀌지 않는다.
- 문서를 비활성화(`INACTIVE`)해도 원본·버전·청크는 삭제되지 않고 검색 대상에서만
  제외된다(soft delete).
